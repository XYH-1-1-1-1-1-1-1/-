# AI 模拟面试与能力提升软件 - 环境配置说明

## 一、系统要求

### 1.1 硬件要求
- CPU: 双核处理器及以上
- 内存：4GB 及以上
- 硬盘：1GB 可用空间
- 网络：需要互联网连接（用于调用大模型 API）

### 1.2 软件要求
- JDK 17 或更高版本
- Maven 3.6 或更高版本
- 现代浏览器（Chrome 90+、Edge 90+ 等，用于语音输入功能）

## 二、环境搭建步骤

### 2.1 安装 JDK 17

**Windows 系统：**
1. 访问 Oracle 官网或 Adoptium 下载 JDK 17
2. 运行安装程序，按照提示完成安装
3. 配置环境变量：
   - 添加 `JAVA_HOME` 指向 JDK 安装目录
   - 在 `Path` 中添加 `%JAVA_HOME%\bin`
4. 验证安装：`java -version`

**Linux 系统：**
```bash
# Ubuntu/Debian
sudo apt update
sudo apt install openjdk-17-jdk

# CentOS/RHEL
sudo yum install java-17-openjdk-devel

# 验证安装
java -version
```

**macOS 系统：**
```bash
# 使用 Homebrew
brew install openjdk@17

# 或者从官网下载安装
```

### 2.2 安装 Maven

**Windows 系统：**
1. 访问 Maven 官网下载最新版本
2. 解压到指定目录
3. 配置环境变量：
   - 添加 `MAVEN_HOME` 指向 Maven 安装目录
   - 在 `Path` 中添加 `%MAVEN_HOME%\bin`
4. 验证安装：`mvn -version`

**Linux/macOS 系统：**
```bash
# Ubuntu/Debian
sudo apt install maven

# macOS (使用 Homebrew)
brew install maven

# 验证安装
mvn -version
```

### 2.3 获取大模型 API Key

本项目使用 OpenRouter 作为大模型 API 提供商（支持多个免费模型）：

1. 访问 [OpenRouter](https://openrouter.ai/)
2. 注册账号并登录
3. 在 Dashboard 中创建 API Key
4. 复制 API Key 备用

**替代方案：**
- [HuggingFace Inference API](https://huggingface.co/docs/api-inference/index)
- [Ollama](https://ollama.ai/) (本地部署，可选)

### 2.4 配置项目

1. **克隆/下载项目代码**

2. **配置 API Key**

   方式一：通过环境变量
   ```bash
   # Windows (CMD)
   set LLM_API_KEY=your-api-key-here
   
   # Windows (PowerShell)
   $env:LLM_API_KEY="your-api-key-here"
   
   # Linux/macOS
   export LLM_API_KEY=your-api-key-here
   ```

   方式二：直接修改配置文件
   编辑 `src/main/resources/application.yml`，将：
   ```yaml
   llm:
     api-key: ${LLM_API_KEY:your-api-key-here}
   ```
   改为：
   ```yaml
   llm:
     api-key: your-actual-api-key
   ```

3. **配置模型（可选）**

   在 `application.yml` 中可以修改使用的模型：
   ```yaml
   llm:
     provider: openrouter
     base-url: https://openrouter.ai/api/v1
     model: google/gemma-2-9b-it:free  # 免费模型
     fallback-models:
       - meta-llama/llama-3-8b-instruct:free
       - mistralai/mistral-7b-instruct:free
   ```

## 三、编译和运行

### 3.1 编译项目

在项目根目录执行：
```bash
mvn clean package -DskipTests
```

### 3.2 运行项目

**方式一：使用 Maven**
```bash
mvn spring-boot:run
```

**方式二：运行打包后的 JAR**
```bash
java -jar target/interview-simulation-1.0.0.jar
```

**方式三：在 IDE 中运行**
- IntelliJ IDEA：右键 `Application.java` -> Run
- Eclipse：右键 `Application.java` -> Run As -> Java Application

### 3.3 访问应用

启动成功后，打开浏览器访问：
```
http://localhost:8080/api/
```

或者直接访问前端页面：
```
http://localhost:8080/api/index.html
```

## 五、常见问题

### 5.1 端口被占用

如果 8080 端口被占用，修改 `application.yml`：
```yaml
server:
  port: 8081  # 改为其他端口
```

### 5.2 API Key 无效

确保 API Key 正确配置，检查：
- API Key 是否正确复制
- 环境变量是否生效
- 配置文件格式是否正确

### 5.3 数据库连接失败

项目使用 SQLite 数据库，数据文件位于 `./data/interview.db`
如果出现问题，可以删除该文件重新启动（会丢失数据）。

### 5.4 语音输入不可用

语音输入使用浏览器 Web Speech API，需要：
- 使用 Chrome、Edge 等支持的浏览器
- 允许浏览器访问麦克风
- 使用 HTTPS 或 localhost 访问
