# 音频分析实现详解

## 实现流程概述

音频分析功能采用**两阶段处理**方式：

```
┌─────────────────────────────────────────────────────────────────┐
│                        音频分析流程                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐      │
│  │  第一阶段    │    │  第二阶段    │    │   结果输出   │      │
│  │  语音识别    │───▶│  情绪分析    │───▶│   JSON 格式   │      │
│  │  (ASR)       │    │  (LLM)       │    │              │      │
│  └──────────────┘    └──────────────┘    └──────────────┘      │
│         │                   │                   │               │
│         ▼                   ▼                   ▼               │
│  音频 → 文本            文本 → 分析         结构化数据          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 详细实现步骤

### 步骤 1：语音识别 (ASR - Automatic Speech Recognition)

**调用阿里云 Paraformer 模型**将音频转换为文本。

```java
// 1. 将音频文件转换为 Base64 编码
byte[] audioBytes = audioFile.getBytes();
String base64Audio = Base64.getEncoder().encodeToString(audioBytes);

// 2. 构建 ASR 请求
JSONObject requestBody = new JSONObject();
JSONObject input = new JSONObject();
input.put("speech", base64Audio);  // Base64 编码的音频

JSONObject taskConfig = new JSONObject();
taskConfig.put("format", "wav");   // 音频格式
taskConfig.put("sample_rate", 16000);
taskConfig.put("enable_punctuation_prediction", true);  // 添加标点
taskConfig.put("enable_inverse_text_normalization", true);  // 文本规范化

requestBody.put("model", "paraformer-realtime-v2");
requestBody.put("input", input);
requestBody.put("task_config", taskConfig);

// 3. 发送 HTTP POST 请求到阿里云 ASR API
HttpPost post = new HttpPost(
    "https://dashscope.aliyuncs.com/api/v1/services/asr/speech-to-text"
);
post.setEntity(new StringEntity(requestBody.toJSONString()));

// 4. 解析响应，获取转录文本
// 响应格式：{"output": {"results": [{"text": "识别结果"}]}}
```

**API 请求示例**:
```bash
curl -X POST \
  https://dashscope.aliyuncs.com/api/v1/services/asr/speech-to-text \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{
    "model": "paraformer-realtime-v2",
    "input": {
      "speech": "base64_encoded_audio_data"
    },
    "task_config": {
      "format": "wav",
      "sample_rate": 16000,
      "enable_punctuation_prediction": true,
      "enable_inverse_text_normalization": true
    }
  }'
```

**API 响应示例**:
```json
{
  "output": {
    "results": [
      {
        "text": "我认为 HashMap 是基于哈希表实现的，它通过 hash 函数将键映射到数组位置..."
      }
    ]
  }
}
```

## 情绪识别原理

千问大模型通过分析以下文本特征来识别情绪：

| 情绪类型 | 识别特征 |
|---------|---------|
| **CONFIDENT (自信)** | 用词肯定、句式完整、逻辑清晰、回答详细 |
| **NERVOUS (紧张)** | 语句断续、重复用词、犹豫表达（"嗯"、"那个"） |
| **CALM (从容)** | 语速平稳、表达自然、条理清晰 |
| **ANXIOUS (焦虑)** | 语句急促、表达担忧、自我怀疑 |
| **EXCITED (兴奋)** | 用词积极、表达热情、详细展开 |
| **NORMAL (平静)** | 情绪波动不大、表达中性 |
| **FOCUSED (专注)** | 回答针对性强、专业术语准确、逻辑严密 |

## 前端集成方式

当前实现中，前端使用**浏览器原生 Web Speech API**进行语音识别：

```javascript
// 前端语音识别代码（index.html）
const SpeechRecognition = window.SpeechRecognition || window.webkitSpeechRecognition;
const recognition = new SpeechRecognition();
recognition.lang = 'zh-CN';
recognition.continuous = true;
recognition.interimResults = true;

recognition.onresult = function(event) {
    let finalTranscript = '';
    for (let i = event.resultIndex; i < event.results.length; i++) {
        if (event.results[i].isFinal) {
            finalTranscript += event.results[i][0].transcript;
        }
    }
    voiceAnswer += finalTranscript;  // 累积转录文本
};

recognition.start();
```

**两种方式对比**:

| 方式 | 优点 | 缺点 |
|-----|------|------|
| **前端 Web Speech API** | 实时转录、无需后端处理音频 | 依赖浏览器支持、精度一般 |
| **后端阿里云 ASR** | 精度高、支持多种格式 | 需要上传音频、有 API 调用成本 |

## 注意事项

1. **API 调用顺序**: 必须先完成 ASR 语音识别，再进行 LLM 情绪分析
2. **错误处理**: ASR 或 LLM 调用失败时，使用默认值避免系统崩溃
3. **响应时间**: 两次 API 调用可能需要 3-10 秒，建议添加加载提示
4. **成本控制**: 每次面试约调用 10 次 ASR + 1 次 LLM 情绪分析
